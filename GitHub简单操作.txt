web端操作
1. 官方网站是https://github.com/ ,sign in
2. 查看自己的项目. 是在登入进去以后右下角有项目名称. 或者 上面导航栏里面用户名. 进入用户空间可以看到


3.详细记录项目页面:
	N commit  一共提交了多少次
	N branch  有多少条分支
	branch:master  这个是当前的分支, 如果在客户端创建分支,并且有commit的话, 这里会有下来菜单  来显示. 详细可以做操作以后再记录
	详细列表是   文件名   提交时候的名字  提交时间
	最下面就是自述文件


客户端操作
	1.	打开GitHub 客户端 如果页面是空得 可以在左下角  点击加号 add 一个 资源库  然后 指定一个本地文件, 可以是一个包, 他会递归检索所有文档
		有了以后, 他会出现一个详细的左右分栏结构的页面
	2. 左边依次是  提交的历史记录 (可以查看版本记录等)   代码的变动(如果是绿红色,说明代码有变化, 金色好像是分支有变化, 等等).点提交写好名字和描述就可以提交到GIT服务器上, 会要求你登入.  然后指定一个项目名称. 
	3.	创建分支结构    第三个条状,  左边的顶点可以拖动上最上面的  分支合并表里 需要按一下merge view ,   左边是要合并的分支, 右边一个是总分支.  然后按一下   记得看一下  changes 栏目是否变成金黄色  然后提交一下  即可.  然后之后的分支可以删除  也可以继续用
